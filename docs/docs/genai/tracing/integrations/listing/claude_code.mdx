# Tracing Claude Code

![Claude Code Tracing via the MLflow plugin](/images/llms/anthropic/claude-code-tracing.png)

[MLflow Tracing](/genai/tracing) provides automatic tracing for Claude Code:

1. **CLI tracing**: Automatically trace interactive Claude Code CLI conversations
2. **SDK tracing**: Trace Claude Agent SDK usage in Python or TypeScript applications

After setup, MLflow automatically captures traces of your Claude Code workflows and logs them to the active MLflow experiment. Traces include:

- User prompts and assistant responses
- Tool usage (file operations, code execution, web searches, etc.)
- Conversation timing and duration
- Tool execution results
- Token usage (input, output, and total tokens)
- Session metadata including working directory and user

## Setup

Claude Code tracing can be configured either for the Claude Code CLI or for programmatic Claude Agent SDK usage.

## Claude Code CLI

### CLI Tracing Setup

Use CLI tracing to automatically capture your interactive Claude Code sessions without depending on the Python environment that launched `claude`.

### Step 1. Start Claude Code

Launch Claude Code in the repo you want to trace.

```bash
claude
```

### Step 2. Install the MLflow Tracing Plugin

Inside Claude Code, add the MLflow plugin marketplace, install the plugin, then reload plugins:

```text
/plugin marketplace add mlflow/mlflow
/plugin install mlflow-tracing@mlflow-plugins
/reload-plugins
```

:::note
If you prefer to manage plugins from the shell instead of inside Claude Code, you can use the corresponding CLI commands:

```bash
claude plugin marketplace add mlflow/mlflow
claude plugin install mlflow-tracing@mlflow-plugins --scope user
```
:::

### Step 3. Run the Setup Command

Ask Claude to run the plugin's setup command:

```text
/mlflow-tracing:setup
```

The setup flow requires:

- Exactly one scope: `--project` or `--user`
- A tracking backend via `--tracking-uri`
- Exactly one experiment selector: `--experiment-id` or `--experiment-name`

For example, you can tell Claude:

```text
Configure MLflow tracing for this repo using:
- scope: --project
- tracking URI: http://localhost:5000
- experiment name: claude-code-traces
```

Supported tracking URIs are:

- `http://...` or `https://...`
- `databricks`
- `databricks://<profile>`

Behind the scenes, the plugin's setup skill runs `mlflow-claude-code setup`. If you pass `--experiment-name` and the experiment does not exist, it creates the experiment and writes the resolved experiment ID into Claude's settings.

### Step 4. Confirm the Configuration

Ask Claude to run the plugin's status command:

```text
/mlflow-tracing:status
```

The installed plugin's status skill reports:

- Whether tracing is enabled
- Where configuration came from: `environment`, `project`, `user`, or `none`
- The active tracking URI
- The active experiment ID and experiment name

### Step 5. Ask Questions as Usual

Once setup is complete, use Claude Code normally:

```text
Can you summarize this repo using subagents?
```

### How It Works

1. Claude Code installs the `mlflow-tracing` plugin from the `mlflow-plugins` marketplace. The marketplace entry resolves to the published npm package `@mlflow/claude-code`.
2. The plugin provides setup and status skills plus a Stop hook.
3. The setup skill writes MLflow tracing settings into either `./.claude/settings.json` or `~/.claude/settings.json`.
4. The Stop hook reads the session transcript and exports a trace to MLflow when the Claude Code session ends.
5. At runtime, configuration precedence is: shell environment variables, then project settings, then user settings.

## Claude Agent SDK

### SDK Tracing Setup

Use SDK tracing when building applications that programmatically use the Claude Agent SDK.

<Tabs>
<TabItem value="python" label="Python" default>

#### Requirements

- MLflow >= 3.5 (`pip install 'mlflow>=3.5'`)
- [Claude Agent SDK](https://docs.claude.com/en/api/agent-sdk/overview) >= 0.1.0 (`pip install claude-agent-sdk>=0.1.0`)

#### Basic Setup

```python
import mlflow.anthropic

# Enable automatic tracing for Claude Agent SDK
mlflow.anthropic.autolog()
```

Once enabled, Claude Agent SDK interactions created through `ClaudeSDKClient` are automatically traced.

:::note
Python SDK tracing currently supports `ClaudeSDKClient`. Directly calling `query()` is not traced.
:::

#### Complete Example

```python
import asyncio
import mlflow
import mlflow.anthropic
from claude_agent_sdk import ClaudeSDKClient

mlflow.anthropic.autolog()
mlflow.set_experiment("my_claude_app")


async def main():
    async with ClaudeSDKClient() as client:
        await client.query("What is the capital of France?")

        async for message in client.receive_response():
            print(message)


if __name__ == "__main__":
    asyncio.run(main())
```

#### Claude Tracing with MLflow Evaluation

You can also use Python SDK tracing with MLflow's evaluation framework:

```python
import asyncio
import pandas as pd
from claude_agent_sdk import ClaudeSDKClient
from typing import Literal

import mlflow
import mlflow.anthropic
from mlflow.genai import evaluate
from mlflow.genai.judges import make_judge

mlflow.anthropic.autolog()


async def run_agent(query: str) -> str:
    async with ClaudeSDKClient() as client:
        await client.query(query)

        response_text = ""
        async for message in client.receive_response():
            response_text += str(message) + "\n\n"

        return response_text


def predict_fn(query: str) -> str:
    return asyncio.run(run_agent(query))


relevance = make_judge(
    name="relevance",
    instructions=(
        "Evaluate if the response in {{ outputs }} is relevant to "
        "the question in {{ inputs }}. Return either 'pass' or 'fail'."
    ),
    feedback_value_type=Literal["pass", "fail"],
    model="openai:/gpt-4o",
)

eval_data = pd.DataFrame([
    {"inputs": {"query": "What is machine learning?"}},
    {"inputs": {"query": "Explain neural networks"}},
])

mlflow.set_experiment("claude_evaluation")
evaluate(data=eval_data, predict_fn=predict_fn, scorers=[relevance])
```

#### Requirements

- Node.js 18+
- [Claude Agent SDK for TypeScript](https://docs.claude.com/en/api/agent-sdk/typescript)
- MLflow TypeScript packages:

```bash
npm install @mlflow/core @mlflow/claude-code @anthropic-ai/claude-agent-sdk
```

#### Basic Setup

Wrap the Claude Agent SDK `query()` function with `createTracedQuery()` and initialize the MLflow TypeScript SDK once at process startup.

```typescript
import * as mlflow from "@mlflow/core";
import { query } from "@anthropic-ai/claude-agent-sdk";
import { createTracedQuery } from "@mlflow/claude-code";

mlflow.init({
  trackingUri: "http://localhost:5000",
  experimentId: "0",
});

const tracedQuery = createTracedQuery(query);
```

#### Complete Example

```typescript
import * as mlflow from "@mlflow/core";
import { query } from "@anthropic-ai/claude-agent-sdk";
import { createTracedQuery } from "@mlflow/claude-code";

mlflow.init({
  trackingUri: "http://localhost:5000",
  experimentId: "0",
});

const tracedQuery = createTracedQuery(query);

const result = tracedQuery({
  prompt: "List the files in this directory",
  options: {
    permissionMode: "bypassPermissions",
  },
});

for await (const message of result) {
  if (message.type === "result") {
    console.log("Result:", message.result);
  }
}
```

:::note
The TypeScript wrapper automatically enables `forwardSubagentText` so nested sub-agent activity can be reconstructed in the MLflow trace.
:::

## Troubleshooting

### Check CLI Status

```text
/mlflow-tracing:status
```

### Common CLI Issues

**Tracing not working:**

- Confirm the marketplace was added and the plugin is installed:
  `claude plugin marketplace list`
  `claude plugin list`
- Reload plugins after installation with `/reload-plugins` or restart Claude Code.
- Ask Claude to run `/mlflow-tracing:status`.
- Verify the correct scope was configured: `./.claude/settings.json` for `--project` or `~/.claude/settings.json` for `--user`.
- Confirm the tracking URI is reachable.

**Missing traces:**

- Claude Code traces are exported from the Stop hook, so traces appear when the Claude Code session ends.
- Check whether shell-level `MLFLOW_*` environment variables are overriding the project or user settings you expected.
- Verify the configured experiment exists and that the resolved experiment ID is valid.

### Disable CLI Tracing

There is no dedicated disable command yet. To disable tracing:

- Uninstall or disable the Claude plugin:
  `claude plugin uninstall mlflow-tracing@mlflow-plugins`
  or `claude plugin disable mlflow-tracing@mlflow-plugins`
- Remove the MLflow tracing environment keys from the relevant `.claude/settings.json`.
- Or override with `MLFLOW_CLAUDE_TRACING_ENABLED=false` in the environment that launches `claude`.

### Common SDK Issues

<Tabs>
<TabItem value="python" label="Python" default>

**No traces appearing:**

- Tracing only works with `ClaudeSDKClient`; direct usage of `query()` is not supported.
- Verify `mlflow.anthropic.autolog()` is called before creating the `ClaudeSDKClient`.
- Check that the tracking URI and experiment are configured correctly.

### Disable SDK Tracing

To disable Python SDK tracing:

```python
mlflow.anthropic.autolog(disable=True)
```

</TabItem>
<TabItem value="typescript" label="JS / TS">

**No traces appearing:**

- Make sure you are calling the wrapped function returned by `createTracedQuery(query)`, not the raw SDK `query()` function.
- Verify `mlflow.init({ trackingUri, experimentId })` runs before the first traced query.
- Confirm you are running in a Node.js environment and that the MLflow tracking server is reachable.
- Consume the query result stream so the trace can be finalized and flushed.
